Visualizing Submodule Information

Goals

Various compound modules in INET contain submodules that display information about their state in the graphical runtime environment. This is of limited use, since one has to enter the compound modules to see this information. Information of important submodules can be visualized at containing network nodes, at the top level of the network. This makes such information conveniently available at glance, without the need to open graphical views or going into compound modules. This example contains a configuration that demonstrates visualization of nodes' IEEE 802.11 channel access state.

About the visualizer

The InfoVisualizer module can visualize information of a submodules for a network node on the top level canvas. The submodules can be selected with the visualizer's modules parameter. This parameter takes full path module names. Wildcards can be used to select a certain submodule name in multiple nodes. Additionally, one can specify multiple submodule names.

The visualization

The network for the simulation is the following:

It contains 3 AdhocHosts arranged in a simple chain. It also contains an IPv4NetworkConfigurator, an Ieee80211ScalarRadioMedium, and an IntegratedVisualizer module. The communicaton ranges of hosts is determined by their radio transmitter power, which is configured so that hosts can only reach adjacent hosts. Hosts' routing tables are set up accordingly, source is configured to send UDP packets to destination via relay.

The visualizer is configured to visualize node's channel access state. This is contained in each node's Contention module. The location of the Contention module is AdhocHost/wlan/mac:

*.visualizer.*.infoVisualizer.modules = "*.*.wlan[0].mac.ctn[0]"
