Mobility visualizer example

Goals - WIP

When wireless nodes roam in the network, it's often difficult to follow their movement. By displaying nodes' movement trajectory, velocity and orientation, we see enough information to track them.

The example consists of 1 simulation model, that demonstrates some features of mobility visualization.

About the visualizer

The MobilityVisualizer displays data about movements of the wireless nodes. The visualizer has three main parameters: Movement trail, velocity and orientation. Movement trail shows the node's recent movement trajectory. We can set its length using the trailLength parameter. Velocity is represented visually by a solid arrow. Its length is proportional to the speed, but we can modify this ratio with a velocity scaling factor called velocityArrowScale. Orientation represented by an arc. Its size can be modified.

Displaying mobility

This example demonstrates, how the customized parameters change the visualization.

We place two Adhochosts on the playground. Because of the mobility settings, they move randomly between the predefined borders. There is one difference between the nodes. The host1 speed is 15mps, the host2 speed is 30mps. Here is the configuration of the mobility:

# Mobility parameters
*.host*.mobilityType = "MassMobility"
*.host*.mobility.changeInterval = 0.5s
*.host*.mobility.changeAngleBy = uniform(-30deg, 30deg)
*.host*.mobility.constraintAreaMinX = 150m
*.host*.mobility.constraintAreaMinY = 100m
*.host*.mobility.constraintAreaMinZ = 0m
*.host*.mobility.constraintAreaMaxX = 700m
*.host*.mobility.constraintAreaMaxY = 500m
*.host*.mobility.constraintAreaMaxZ = 0m

*.host1.mobility.speed = 15mps
*.host2.mobility.speed = 30mps

The visualization of mobility is enabled, because the displayMovements parameter is true by default. It is really confusing, if we run the simulation now:

We display the movement trail, the velocity vector and the orientation arc of the node. Here is the configuration:

# Visualizer parameters
# Movement trail
*.visualizer.*.mobilityVisualizer.displayMovementTrails = true

# Velocity vector
*.visualizer.*.mobilityVisualizer.displayVelocities = true

# Orientation
*.visualizer.*.mobilityVisualizer.displayOrientations = true
Now it is easy to follow the nodes' movement:

It could happen that, we want to customize the visualization e.g. we want to highlight any parameter. Now we specifiy the parameters of velocity arrow:

# Velocity vector
*.visualizer.*.mobilityVisualizer.displayVelocities = true
*.visualizer.*.mobilityVisualizer.velocityArrowScale = 2
*.visualizer.*.mobilityVisualizer.velocityLineColor = "blue"
*.visualizer.*.mobilityVisualizer.velocityLineStyle = "solid"
*.visualizer.*.mobilityVisualizer.velocityLineWidth = 2
