Mobility visualizer example

Goals

When wireless nodes roam in a network, it's often difficult to follow their movement. Using MobilityVisualizer we can track their movement trajectory and their speed. The visualizer helps to determine nodes' orientation too. It is really important if the node is a directional antenna.

The example consists of 7 simulation models, each demonstrating different features of mobility visualization.

About the visualizer

The MobilityVisualizer displays data about wireless nodes movements such as movement trail, velocity vector and orientation. The visualizer has three main parameters: Movement trail, velocity and orientation. Movement trail shows the node's recent movement trajectory. We can set its length using the trailLength parameter. Velocity is represented visually by a solid arrow. Its length is proportional to the speed, but we can modify this ratio with a velocity scaling factor called velocityArrowScale. Orientation represented by an arc. Its size can be modified.
