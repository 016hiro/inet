Mobility Visualization Example

Goals

When wireless nodes roam in the network, it's often difficult to follow their movement. By displaying nodes' movement trajectory, velocity and orientation, we can see enough information to track them.

The example consists of one simulation model, that demonstrates important features of mobility visualization.

About the Visualizer

The MobilityVisualizer displays data about the movements of wireless nodes. The visualization is enabled by the displayMovements parameter, which is true by default.

The visualizer has three main features: Movement trail, velocity vector and orientation. By default, these features are turned off. Each of these features has a parameter, which enables its visualization by setting it to true. These parameters are the displayMovementTrails, the displayVelocities and the displayOrientations.

Movement trail shows the node's recent movement trajectory. We can set its length using the trailLength parameter. When the trail's length is longer than the trailLength's value, the end of the line starts to fade out. The trail's color is different for each node, because of the movementTrailLineColor = "auto" parameter. This option makes the nodes' traceability easier.

Velocity is represented visually by a solid arrow. Its length is proportional to the speed. By modifying the velocityArrowScale parameter we multiply the arrow's size. The arrow's direction coincides with the movement's direction.

Orientation is represented by an arc. The arc's size is specified by the orientationArcSize parameter. This value is the relative size of the arc compared to a full circle. Its default value is 0.25, i.e. a quarter or a circle.

Displaying Mobility

This example demonstrates, how the customized parameters change the visualization.

We place two Adhochosts on the playground. Because of the mobility settings, they move randomly between the predefined borders. There is one difference between the nodes. The host1's speed is 15 mps, host2's speed is 30 mps.

When we start the simulation with the visualization's default settings, we can see nodes which move randomly on the playground. It is really confusing we can't trace their movement:

We display the movement trail, the velocity vector and the orientation arc of the node. Here is the configuration:

# Movement trail
*.visualizer.*.mobilityVisualizer.displayMovementTrails = true

# Velocity vector
*.visualizer.*.mobilityVisualizer.displayVelocities = true

# Orientation
*.visualizer.*.mobilityVisualizer.displayOrientations = true
Now it is easy to follow the nodes' movement:

Each node has a movement trail, a velocity vector and an orientation arc. The trails line colors are different, because the trail's line color is "auto". The host2's velocity vector is longer, because its speed is twice than the host1's speed. The orientation arcs are turned to the movement's direction in every moment.

Sometimes it's useful to highlight a certain feature of the visualization. For example, we highlight the velocity arrow by customizing its appearance parameters:

# Velocity vector
*.visualizer.*.mobilityVisualizer.displayVelocities = true
*.visualizer.*.mobilityVisualizer.velocityArrowScale = 2
*.visualizer.*.mobilityVisualizer.velocityLineColor = "blue"
*.visualizer.*.mobilityVisualizer.velocityLineStyle = "solid"
*.visualizer.*.mobilityVisualizer.velocityLineWidth = 2

Here is the result of these settings:

The arrow turned blue because of the velocityLineColor is set to blue and the arrow has become two times longer and it has became twice as wide than the default velocity vector.
