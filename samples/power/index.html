Power Consumption Example

Goals

This example demonstrates consumption, storage and generation of power in mobile wireless nodes. The hosts will send out ping requests, which will deplete their power storage, while their power generators will charge them.

The model

The network

The simulation contains mobile hosts (type adhochost), the number of which can be specified in omnetpp.ini. The default number specified is 20. They are randomly placed on the playground, and don't move around.

Configuration and behavior

The hosts 1 through 20 are set to periodically send out ping requests to host 0. The radio transmissions will deplete the hosts' energy storage.

Each host has a nominal energy storage capacity of 0.05 Joules. The charge they contain at the beginning of the simulation is randomly selected between 0 and the nominal capacity. Hosts shut down when their energy storage reaches 0, and restart when their energy storage charges to half of the nominal capacity, that is 0.025 Joules.

The hosts contain an AlternatingEnergyGenerator. This alternates between generation and sleep modes. It starts in generation mode, and generates the amount of power that is specified by its powerGeneration parameter (now set to 1 mW). Sleep mode, it generates no power. It stays in each mode for the duration specified by the generationInterval and sleepInterval parameters. These are set to a random value with a mean of 10s.

Host 0 will act as the target for the ping target for all the other hosts.
